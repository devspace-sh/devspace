`command` and `args` can be used to replace a pods command and args in development mode.

For example:
```yaml
deployments:
  my-deployment:
    helm:
      values:
        containers:
          - image: ubuntu

dev:
  my-dev:
    imageSelector: ubuntu
    command: ["sleep"]
    args: ["100000"]
```

## Configuration

### `command`

A string array that specifies the container command to run on startup.

### `args`

A string array that specifies arguments that should be passed to the container command on startup.
